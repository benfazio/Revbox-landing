Agency Update Related Records Function



The Agency Update Related Records Function allows users to change the related agency in an agency code and the function will automatically update the agency/account in related Agency Report(s) and Agency LOB Report(s).

Agency Update Related Records Architecture



  1. The workflow is triggered any time the agency in an Agency Code is modified and the Agency_Report_ID of the changed Agency Code is passed to the Zoho Workflow Rule named AgencyUpdateRelatedRecords. 
  2. The AgencyUpdateRelatedRecords workflow rule calls the lambda function named Agency_Update_Related_Records.
  3. The  Agency_Update_Related_Records lambda function updates the related Agency Reports and Agency LOB Reports with the new Agency/Account. 


Additional Notes

  • AgencyUpdateRelatedRecords Workflow Rule
  • The workflow rule is triggered anytime the Agency field in an Agency Code is modified. 
  • The changed agency code's id is assigned to a variable, a getRecordById is called in the Deluge Function using the Agency_Code_ID. From this we get the new Agency/Account ID and the Account ID variable and the Agency Code ID Variable are appended to the URL used to call the lambda function.
  • Agency Update_Related_Records Lambda Function
  • The lambda function is triggered when the AgencyUpdateRelatedRecords workflow rule calls it.
  • The function takes the changed agency code's id and the new account id as a query string parameter and uses it to delete all the related records.
  • Firstly, the changed agency code's id is used to search for all the related Agency Reports. It stores all the Agency Reports' ids and updates all related Agency Reports with the new Account ID that was sent to the lambda function.  
  • Once all of the related Agency Reports are updated, the second step is updating all of the related Agency LOB Reports. It searches and stores all of the Agency LOB Reports' ids. Then it updates the related Agency LOB Reports.

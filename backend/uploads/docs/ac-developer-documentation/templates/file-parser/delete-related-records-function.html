Delete Related Records Function


The Delete Related Records Function allows users to delete a carrier report and have the related reports automatically deleted.


Delete Related Records Architecture

  1. The carrier report's id that was deleted from the CRM is passed to the Zoho Workflow Rule named DeleteRelatedRecords.
  2. The DeleteRelatedRecords workflow rule calls the lambda function named Delete_Related_Records.
  3. The Delete_Related_Records lambda function deletes the related Agency Reports, AC LOB Total Reports, and Member Data LOB Reports. 


Additional Notes

  • DeleteRelatedRecords Workflow Rule
  • The workflow rule is triggered when a carrier report is deleted. 
  • The deleted carrier report's id is assigned to a variable, and the variable is appended to the URL used to call the lambda function.
  • Delete_Related_Records Lambda Function
  • The lambda function is triggered when the DeleteRelatedRecords workflow rule calls it.
  • The function takes the deleted carrier report's id as a query string parameter and uses it to delete all the related records.
  • Firstly, the deleted carrier report id is used to search for all the related Agency Reports. It stores all the Agency Reports' ids and deletes the reports at a maximum of a hundred at a time.  
  • Once all of the related Agency Reports are deleted, the second step is deleting all of the related AC LOB Total Reports. It searches and stores all of the AC LOB Total Reports' ids. Then it deletes the reports at a maximum of a hundred at a time. The AC LOB Total Reports' ids are returned to the main function.
  • The AC LOB Total Reports' ids are then used to delete the Member Data LOB Reports. Then it deletes the reports at a maximum of a hundred at a time.

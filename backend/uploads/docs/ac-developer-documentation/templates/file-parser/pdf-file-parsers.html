PDF File Parsers


The AC File PDF Parser is a solution that we have developed to automate the process of entering new PDF carrier data that the AC receives monthly. This article will outline the architecture, tools, and additional notes that were needed to create this feature.

PDF Parser Architecture (Serverless)

 

  1. Frontend calls the Presigned URL function.
  2. The Presigned URL function gets a key from the textractkvtable-us-east-1-907387566050 bucket that allows a file to be uploaded from the frontend. It also edits a JSON file that contains the file's info.
  3. Finally, it returns the presigned URL and key to the frontend.
  4. Then we upload the file to the textractkvtable-us-east-1-907387566050 bucket using the presigned URL.
  5. The file that got uploaded invokes a lambda function named Job Lambda.
  6. Job Lambda uses Textract to get the file parsed.
  7. The Textract response is then stored in the textractkvtable-us-east-1-907387566050 bucket.
  8. Textract invokes Amazon SNS.
  9. Amazon SNS invokes ParseLambda.
  10. ParseLambda invokes Textract and uses it to parse formatted excel table files.
  11. The formatted excel table files are stored in the textractkvtable-us-east-1-907387566050 bucket.
  12. After ParseLambda is finished, the Parser function is invoked.
  13. The Parser function takes the excel files from the textractkvtable-us-east-1-907387566050 bucket.
  14. It grabs agency codes and account ids from the Zoho CRM. Then parses the files and then uploads the data to the Zoho CRM.
  15. The Most Recent Trigger Function is then invoked.
  16. The Most Recent Trigger function will trigger the Most Recent function by sending commands to run a batch file.
  17. It will then modify the carrier's records to the most recent reports.
  18. Once the Most Recent function is done running, the Email Notification function is invoked. 
  19. The function will send a success email to the customer's email.

Foremost Bristol West Parser

The Foremost Bristol West Parser is a special case since the PDFs are 500+ pages. 

  1. PDF file is dropped in the frontend and uploaded to s3.
  2. JobLambda is invoked and invokes a lambda function called EC2_Foremost_Bristol_West_Trigger_Pt1.
  3. The lambda function invokes a batch file in the AC File Parser EC2 instance.
  4. The batch file then starts up a script called ForemostBristolWestPt1.py.
  5. The script splits the file into separate PDFs containing 10 pages (besides the first file which only contains 2 agencies). It creates the Carrier Report and LOB Total Reports. The IDs of these reports uploads to the foremost.json in s3. It also reads in the agency names and codes and uploads the data to codes.json in s3. The filename and folder name are stored in the users.json in s3.
  6. Every split file is uploaded to s3 for JobLambda and ParseLambda to parse with Textract.
  7. After every file is parsed by Textract. You have to manually index the list of folder names and code in the ForemostBristolWestParser lambda function.
  8. Make sure that the indexes of the codes match up to the pages based on the list of folder names. There are instances where a file gets skipped so make sure to check if all of the folders are in the s3 bucket. You will likely have to run this function twice due to the 15 minute limit of running a lambda function.

  • Presigned URL Function
  • There are two Presigned URL functions. One is for PDFs, and the other is for excel.
  • The function generates a presigned URL that will allow us to drop a file in the corresponding S3 bucket and a key, which is the filename. 
  • It also edits a JSON file in the textractkvtable-us-east-1-907387566050 bucket and inserts the carrier, filename, key, and user.
  • If there is an error in the Presigned URL functions, then it will send out an email notification to the customer and developer.
  • AC_File_Parser Function
  • This function invokes the PDF and excel parsers in the EC2 server with a command that runs a batch file.
  • In the command, it takes in five parameters. They are the carrier, filename, key, uploaded file name, and user, which are obtained from the users.json file in s3. 
  • Most Recent Function
  • This function is located inside the EC2 Server and is invoked by a lambda function named "mostRecentTrigger".
  • It grabs all the carriers' carrier reports and finds the newest report. Then it takes the carrier report id and iterates through every agency report. If the carrier report id matches, the report updates to be most recent. The AC LOB Total reports also updates by checking if the carrier report ids match. However, the Member LOB reports updates using the AC LOB Total report ids. 
  • Email Notification Function
  • There are two email notification functions.
  • If the parsing is successful, the success notification notifies the customer an email stating success.
  • If the parsing is unsuccessful, the function notifies the customer and developers. The email specifies where the error occurs in the email sent to the developer.
  • Textract
  • Textract is a machine learning (ML) service that automatically extracts text, handwriting, and data from scanned documents.
  • We use Textract to parse PDF files and extract the tables from the file. The tables are each separated into an excel file and stored in a folder named after the key. The folder location is in a folder called "tables/" in the textractkvtable-us-east-1-907387566050 bucket.
  • Amazon SNS
  • Amazon Simple Notification Service is a web service that coordinates and manages the delivery or sending of messages to subscribing endpoints or clients.
  • We use Amazon SNS to trigger ParseLambda once Textract finishes.

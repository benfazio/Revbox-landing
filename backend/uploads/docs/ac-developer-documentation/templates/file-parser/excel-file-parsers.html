Excel File Parsers


The AC File Excel Parser is a solution that we have developed to automate the process of entering new spreadsheet carrier data that the AC receives monthly. This article will outline the architecture, tools, and additional notes that were needed to create this feature.


Excel Parser Architecture (Serverless)

 

  1. Frontend calls the Presigned URL function. 
  2. The Presigned URL function generates a presigned URL and key from the excel-parser-files bucket that allows a file to be uploaded from the frontend. The key will be used to call the Parser function and Most Recent function.
  3. It then edits a JSON file that contains the file's info in the textractkvtable-us-east-1-907387566050 bucket.
  4. Finally, it returns the pre-signed URL and key to the frontend.
  5. Then we upload the file to the excel-parser-files bucket using the presigned URL.
  6. After the file is uploaded, we call the parser function that is based on the carrier file that was uploaded.
  7. The Parser Function then pulls the file from the excel-parser-files bucket.
  8. It grabs agency codes and account ids from the Zoho CRM. Then parses the files and then uploads the data to the Zoho CRM.
  9. After all the records are uploaded, it invokes the Most Recent Trigger function.
  10. The Most Recent Trigger function will invokes the Most Recent function by sending commands to run a batch file.
  11. The Most Recent function will then modify the carrier's records to the most recent reports.
  12. Once the Most Recent function is done running, the Email Notification function is invoked.
  13. The function will send a success email to the customer's email.

  6. After the file is uploaded, we call a lambda function.

  8. The Parser Function then pulls the file from the excel-parser-files bucket.
  9. It grabs agency codes and account ids from the Zoho CRM. Then parses the files and then uploads the data to the Zoho CRM.
  10. After all the records are uploaded, it invokes the Most Recent Trigger function.
  11. The Most Recent Trigger function will invoke the Most Recent function by sending commands to run a batch file.
  12. The Most Recent function will then modify the carrier's records to the most recent reports.
  13. Once the Most Recent function is done running, the Email Notification function is invoked.
  14. The function will send a success email to the customer's email.

  • Presigned URL Function
  • There are two Presigned URL functions. One is for PDFs, and the other is for excel.
  • The function generates a presigned URL that will allow us to drop a file in the corresponding S3 bucket and a key, which is the filename. 
  • It also edits a JSON file in the textractkvtable-us-east-1-907387566050 bucket and inserts the carrier, filename, key, and user.
  • If there is an error in the Presigned URL functions, then it will send out an email notification to the customer and developer.
  • Excel Parsers (Serverless)
  • For each carrier, there is a lambda function that parses the carrier's file.
  • The serverless excel parsers are invoked by a frontend call using an API endpoint. In the excel parser, it passes the filename, carrier, key, and user as query string parameters in the API endpoint URL. 
  • When the function executes, the file is read from the excel-parser-files bucket using the key. The parser then grabs agency codes and account ids from the Zoho CRM with a COQL query function. A report date is created and records are uploaded to the Zoho CRM as the file is getting parsed.
  • If the function fails at any point, an email will go out to the customer and developers.
  • Excel Parsers (Server)
  • For each carrier, there is a function in the EC2 server that parses the carrier's file.
  • Both parsers are invoked by a lambda function named AC_File_Parser.
  • Excel Parsers
  • When the function executes, the file is read from the excel-parser-files bucket using the key. The parser then grabs agency codes and account ids from the Zoho CRM with a COQL query function. A report date is created and records are uploaded to the Zoho CRM as the file is getting parsed.
  • If the function fails at any point, an email will go out to the customer and developers.
  • The reason these excel parsers are in the server is the runtime limit for a lambda function is 15 minutes.
  • If the function fails at any point, an email will go out to the customer and developers.
  • AC_File_Parser Function
  • This function invokes the PDF and excel parsers in the EC2 server with a command that runs a batch file.
  • In the command, it takes in five parameters. They are the carrier, filename, key, uploaded file name, and user, which are obtained from the users.json file in s3. 
  • Most Recent Function
  • This function is located inside the EC2 Server and is invoked by a lambda function named "mostRecentTrigger".
  • It grabs all the carriers' carrier reports and finds the newest report. Then it takes the carrier report id and iterates through every agency report. If the carrier report id matches, the report updates to be most recent. The AC LOB Total reports also updates by checking if the carrier report ids match. However, the Member LOB reports updates using the AC LOB Total report ids. 
  • Email Notification Function
  • There are two email notification functions.
  • If the parsing is successful, the success notification notifies the customer an email stating success.
  • If the parsing is unsuccessful, the function notifies the customer and developers. The email specifies where the error occurs in the email sent to the developer.
